### 2.2 跨数据集性能差异分析

```latex
\section{讨论}

\subsection{跨数据集性能差异分析}

实验结果显示，TimesNet系列模型在PSM数据集上表现优异（F1=0.9735），但在RuralVoltage数据集上性能显著下降（F1=0.6059），差距达37.8\%。这一现象值得深入分析。

\textbf{数据特性差异}：PSM数据集来自服务器监控系统，包含25维特征和132,481条训练样本；RuralVoltage数据集仅有16维特征和10,000条训练样本。特征维度和样本量的差异直接影响模型的学习能力。更重要的是，两个数据集的时序模式存在本质区别：服务器监控数据的周期性相对规律（如CPU利用率的日周期），而电压数据的周期性受多种因素影响（负荷变化、天气、用户行为），模式更加复杂。

\textbf{异常类型差异}：PSM数据集的异常主要表现为传感器故障和参数越限，异常模式相对单一；RuralVoltage数据集包含5类异常（欠压、过压、骤降、谐波、不平衡），各类异常的表现形式和持续时间差异显著。多类型异常增加了模型学习的难度。

\textbf{异常比例差异}：PSM数据集的异常比例约为27.8\%，而RuralVoltage约为10\%。较低的异常比例意味着模型在训练过程中接触到的异常模式较少，可能导致对罕见异常的识别能力不足。

\textbf{启示}：跨领域应用时序异常检测模型时，需要充分考虑目标领域的数据特性。在电压监测场景下，可能需要更大规模的训练数据、更长的观测序列或更精细的特征工程来提升性能。
```

### 2.3 召回率偏低问题分析

```latex
\subsection{召回率偏低问题分析}

在RuralVoltage数据集上，所有模型的精确率（0.75--0.90）明显高于召回率（0.19--0.47），表明模型存在较多漏检。这一现象的原因和应对策略值得讨论。

\textbf{原因分析}：

（1）\textit{阈值设定偏保守}：当前采用的阈值基于训练集重构误差的统计分布，在精确率和召回率之间选择了偏向精确率的平衡点。在实际应用中，可根据业务需求调整阈值以提高召回率。

（2）\textit{异常模式多样性}：RuralVoltage数据集包含多种异常类型，部分异常（如轻微的三相不平衡）的信号强度较弱，与正常数据的边界模糊，模型难以准确识别。

（3）\textit{数据集规模限制}：10,000条训练样本可能不足以覆盖所有正常模式的变化，导致部分正常变化被模型误认为需要重构的模式，而真正的异常因与这些"过学习"的模式相似而被漏检。

（4）\textit{时序上下文不足}：seq\_len=100的设置可能无法捕获某些需要更长上下文才能识别的异常模式，如缓慢演变的电压偏移。

\textbf{改进策略}：

（1）\textit{自适应阈值}：根据不同时段或不同异常类型设置差异化阈值，如对电压骤降采用较低阈值以提高敏感度。

（2）\textit{多模型集成}：结合多个模型的预测结果，通过投票或加权机制提高召回率，同时控制精确率下降幅度。

（3）\textit{代价敏感学习}：在训练过程中对漏检（假负例）赋予更高的惩罚权重，引导模型向高召回率方向优化。

（4）\textit{增量学习}：持续收集新的异常样本，通过增量学习扩展模型对罕见异常的识别能力。
```

### 2.4 VoltageTimesNet表现分析

```latex
\subsection{VoltageTimesNet表现分析}

VoltageTimesNet是本研究提出的融合电网领域知识的改进模型，但在RuralVoltage数据集上的表现（F1=0.3123）不及原始TimesNet（F1=0.6059），这一结果需要深入分析。

\textbf{根因分析}：

VoltageTimesNet的核心设计是混合周期发现策略，将FFT自适应周期与电网预设周期（60s、300s、900s、3600s）相结合。然而，在seq\_len=100的配置下，有效周期需满足条件：周期长度$\leq$seq\_len/2=50。这意味着预设的四个周期中仅有60s（按离散采样点计算可能为50-60）处于有效边界，其余300s、900s、3600s周期均无法被利用。

在这种情况下，混合策略实际上退化为"FFT + 少量无效预设"，预设周期不仅未能提供有益信息，反而可能引入噪声，干扰模型的周期建模。

\textbf{与PSM数据集的对比}：

在PSM数据集上，VoltageTimesNet（F1=0.9731）与TimesNet（F1=0.9735）性能相当。这是因为PSM数据集的时序模式与电网周期无关，预设周期不发挥作用时，模型退化为纯FFT模式，与TimesNet等效。

\textbf{改进方向}：

（1）\textit{延长序列长度}：当seq\_len$\geq$720时，60s和300s周期均可生效；当seq\_len$\geq$1800时，所有预设周期均有效。在计算资源允许的情况下，延长序列长度是最直接的改进方案。

（2）\textit{自适应预设周期}：根据序列长度动态调整预设周期集合，仅保留有效周期，避免引入无效周期的噪声。

（3）\textit{多尺度融合}：在不同序列长度上分别运行模型，然后融合预测结果，兼顾短期和长期周期特征。

（4）\textit{周期有效性检验}：在训练过程中评估各预设周期的实际贡献，对贡献较小的周期降低权重或剔除。

\textbf{结论}：VoltageTimesNet的设计理念——融合领域知识——是有价值的，但需要合适的序列长度配置才能发挥作用。当前seq\_len=100的设置限制了预设周期策略的有效性，这一发现为后续工作指明了方向。
```

### 2.5 研究局限与改进方向

```latex
\subsection{研究局限与改进方向}

本研究在方法验证和实验分析方面取得了一定成果，但仍存在以下局限：

\textbf{数据集局限}：
\begin{enumerate}
	\item RuralVoltage数据集为基于国标规则生成的合成数据，可能无法完全反映真实农村电网的复杂性。后续工作需在真实电网数据上进行验证。
	\item 数据集规模（10,000条训练样本）相对较小，可能限制了模型学习复杂模式的能力。
	\item 异常类型虽然涵盖了主要电压质量问题，但未包含复合异常（如同时出现骤降和谐波）的情况。
\end{enumerate}

\textbf{方法局限}：
\begin{enumerate}
	\item 基于重构的方法假设异常数据难以被准确重构，但对于某些与正常模式相似的异常可能效果不佳。
	\item 单一阈值机制难以适应不同异常类型的差异化检测需求。
	\item 当前方法仅进行异常检测，未涉及异常分类，无法直接判断异常类型。
\end{enumerate}

\textbf{实验局限}：
\begin{enumerate}
	\item 由于时间和资源限制，消融实验未能完全覆盖所有设计选择。
	\item 未与最新的异常检测方法（如2024年发表的模型）进行对比。
	\item 未评估模型在在线推理场景下的实时性能。
\end{enumerate}

\textbf{未来工作方向}：
\begin{enumerate}
	\item \textit{真实数据验证}：与电力公司合作获取真实农村电网数据，验证模型的实际应用效果。
	\item \textit{异常分类扩展}：在检测基础上增加分类模块，实现异常类型的自动判别。
	\item \textit{在线部署优化}：针对边缘计算场景进行模型压缩和推理优化，实现毫秒级实时检测。
	\item \textit{多模态融合}：结合电压数据与其他信息（如天气、负荷预测），提升检测的准确性和可解释性。
	\item \textit{自监督预训练}：利用大规模无标签电压数据进行预训练，提升模型的泛化能力。
\end{enumerate}
```
